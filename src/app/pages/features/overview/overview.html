Overview

There are 2 main concepts to understand about @ks89/angular-modal-gallery.

  • Components: Carousel, Plain gallery and Modal gallery are the main features of this library. A detailed description is available in dedicated sections.
  • Configuration: using an object of type LibConfig you can configure the behaviour of all components. There is only an object to configure all components.

Components

  • Carousel: available via <ks-carousel></ks-carousel>
  • Modal gallery: available injecting the ModalGalleryService into your component
  • Plain gallery: available via <ks-plain-gallery></ks-plain-gallery>

Carousel


It contains a list of scrollable images (full width or fixed width) also with previews. By default images scroll automatically. Users can configure this component with many parameters, also they can project HTML content into the current carousel image.

For more info check Carousel's documentation


Modal gallery


  1. Current image
  2. Side previews
  3. Upper buttons
  4. Dots
  5. Previews
  6. Background (semi-transparent)
  7. Modal gallery (the main component)

For more info check Modal gallery's documentation


Plain gallery


It contains a list of thumbnails that users can click to open modal gallery. Layouts are configurable and users could also use a custom implementation to draw everything they want.

For more info check Plain gallery's documentation


LibConfig

The most important interface used to configure components is LibConfig. In case of plain-gallery and carousel you have to pass an object of type LibConfig as component input. Instead, in case of modal-gallery it will be a parameter of a service function.

LibConfig is an object with many optional properties. However, not all of them are used by all components. Here you can find a table to know which of these properties can impact on components
